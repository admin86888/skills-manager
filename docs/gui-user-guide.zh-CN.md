# Skills Manager GUI 使用说明

本文依据当前版本的界面、前端调用和后端文件操作整理。它说明“点击后会做什么”，重点区分中央技能库、Agent 全局目录和项目目录。不同操作系统、Agent 版本和自定义路径会改变实际磁盘路径。不要假定确认框会列出路径或完整数量；执行删除、禁用或改路径前，应先从设置页记录 Agent Skills 路径，并在目标工作区核对对象和数量。

> **重要：** GUI 中的“移除”不总是只删除软链接。受管 target 可能采用 `symlink` 或 `copy` 模式；工作区还会显示非受管本地内容。删除 copy target 会删除复制出的目录或文件，删除非受管本地 skill 甚至可能删除真实目录。先确认对象类型和作用范围。

> **共享路径风险：** 多个 Agent 可能使用同一个全局或项目 Skills 根；内置 Agent 也存在共享默认路径。部分单 Agent 删除、禁用、改路径和项目删除流程按物理路径直接移除，不会先确认该路径是否仍被其他 Agent 记录引用。因此，一个 Agent 的操作可能让共享该路径的其他 Agent 同时失去 target，并留下与磁盘不一致的管理记录。删除前在设置和项目详情中比较各 Agent 的实际路径，不能只看 Agent 名称。

## 1. 核心术语

| 术语 | 含义 | 常见位置 |
|---|---|---|
| 中央 skill | Skills Manager 保存和维护的技能源。安装、更新、标签和备份主要围绕它进行 | 中央库，默认位于 Skills Manager 的仓库目录 |
| target | 从中央 skill 同步到某个 Agent 或项目 Skills 目录的目标项 | 如 `~/.codex/skills/<name>` 或 `<project>/.claude/skills/<name>` |
| 受管 skill | Skills Manager 知道来源和 target 记录的 skill | 工作区中由中央库添加或通过扫描导入的项 |
| 非受管本地 skill | 已在 Agent/项目目录中存在，但尚未纳入中央库管理的内容 | Agent 工作区或项目工作区扫描结果 |
| Preset | 一组 skill，并可在详情中配置 Agent 开关；工作区 PresetBar 当前不按这些开关过滤 | 侧边栏、我的 Skills、各工作区顶部 |
| active Preset | 后端记录的“最近应用到默认全局目标”的 Preset。启用 Agent、改路径等重建动作以它为源 | 不等于侧边栏选择，也不等于尚未重启生效的默认启动 Preset |
| 默认启动 Preset | 设置页保存的启动选择；下次 GUI 启动时会成为 active Preset并执行全局同步 | 设置页 |
| 正在查看的 Preset | 侧边栏当前选中、保存在浏览器本地存储中的 Preset，只控制浏览和编辑上下文 | 不改变 active Preset或默认启动 Preset |
| 全局工作区 | 管理 Agent 用户级 Skills 目录 | 对所有项目生效，例如 `~/.codex/skills` |
| 项目工作区 | 管理某个项目内的 Agent Skills 目录 | 只对该项目生效 |
| `symlink` 模式 | target 是指向中央 skill 的软链接 | 中央内容变化后，Agent 通常直接看到变化 |
| `copy` 模式 | target 是中央 skill 的独立副本 | 中央内容和 target 不会因文件引用自动保持一致 |

Preset 不是互斥配置，也不是持续强制的“期望状态”。只有各工作区顶部的 **PresetBar** 点击会执行一次批量磁盘操作：缺什么就补什么；再次点击已完整生效的 Preset，则移除该 Preset 在当前作用域中的成员 target。侧边栏点击 Preset 只切换“正在查看的 Preset”并导航到“我的 Skills”，不会直接添加或删除 target。多个 Preset 可以叠加，独立添加的 skill 也可同时存在。

## 2. 界面导航

侧边栏提供以下入口：

- **首页**：查看中央库、Agent、Preset 和近期活动的概览，进入常用工作流。
- **我的 Skills**：管理中央 skill、标签、Preset 成员、更新和文档预览。
- **安装 Skills**：从市场、Git、本地目录、压缩文件或现有 Agent 目录导入。
- **全局工作区**：查看全部编码 Agent，或进入指定 Agent 的用户级 Skills 目录。
- **Lobster Agent 工作区**：查看 Lobster 类 Agent 的全局 Skills 目录；作用域和普通全局工作区相同，但导航分组不同。
- **项目工作区**：管理已登记项目及其 Agent 专属目录。
- **备份**：连接 Git 远端、查看同步状态、处理冲突和恢复快照。
- **设置**：配置中央库、Agent 路径、同步模式、平台启用状态、外观、代理、更新、托盘和诊断。

### 2.1 命令面板与快捷键

- `Cmd/Ctrl+K`：打开命令面板，可搜索中央 skill、Preset、项目和页面入口。选择 skill 会打开“我的 Skills”详情；选择项目会打开项目页。
- 在命令面板选择 Preset 只会切换“正在查看的 Preset”并进入“我的 Skills”，不会应用该 Preset，也不会创建或删除 target。
- `Cmd/Ctrl+,`：打开设置。光标位于输入框、文本区或可编辑区域时不触发。
- `Cmd/Ctrl+R`：刷新应用数据，不等于重新启动 Agent；光标位于输入框、文本区或可编辑区域时不触发。

### 2.2 系统托盘

启用托盘后，菜单显示中央 skill 数、已连接编码 Agent 数、可用更新数和 Preset 状态，并提供打开应用、检查 skill 更新、打开中央 Skills 目录和退出。

托盘 Preset 与工作区 PresetBar 一样采用补缺/移除语义：`active` 点击后移除，`partial` 或 `inactive` 点击后补齐。作用域固定为全部“已启用且已安装”的编码 Agent，不包含 Lobster Agent，也不读取 Preset 详情中的 Agent 开关。点击后立即批量执行，没有二次确认；先根据菜单的 `synced/total` 或勾选状态确认方向。

## 3. 推荐日常工作流

1. 在 **安装 Skills** 中把 skill 安装或导入中央库。
2. 在 **我的 Skills** 中检查名称、说明、标签和内容；需要批量复用时加入 Preset。
3. 打开目标 Agent 的 **全局工作区**，通过 Preset 或“添加 Skills”建立全局 target。
4. 只希望某个项目使用时，改在 **项目工作区** 添加，不要添加到全局工作区。
5. 删除前确认对象是中央 skill、受管 target 还是非受管本地目录。
6. 需要多设备同步或版本恢复时连接 **备份**。
7. 遇到计数或目录不一致时，先刷新工作区，记录正在查看的 Preset、默认启动 Preset和最近一次 Preset动作，再核对平台启用状态、路径覆盖和同步模式。GUI 无法直接显示 active Preset时，不要据此执行批量删除。

## 4. 首页

首页用于快速了解当前状态和进入其他页面。概览计数来自中央库、已识别 Agent、Preset 和活动记录；它不等于某个 Agent 启动时实际加载的 skill 数量。

常见差异来源：

- 同一个中央 skill 可同时为多个 Agent 建立 target。
- Agent 目录可能包含非受管本地 skill。
- 工作区 PresetBar 统计的是“Preset skills × 当前页面 Agents”，工作区总数则是当前目录内容。
- Agent 自身可能递归扫描、去重、忽略无效目录，或还未刷新运行时缓存。

首页只提供概览和入口。判断实际状态时，以对应 Agent/项目工作区显示的目标目录内容为准；判断 Agent 运行时加载结果时，还需在 Agent 内重新启动或刷新后验证。

## 5. 安装 Skills

安装页支持以下来源：

### 5.1 skills.sh 市场

浏览或搜索市场条目，选择后安装到中央库。市场可用性受网络、代理和来源服务影响。安装完成不代表已同步到任何 Agent；还需在工作区添加或应用 Preset。

### 5.2 Git URL

输入 Git 仓库 URL 后先预览可识别的 skill，再勾选一个或多个条目安装。仓库根目录不一定就是 skill；预览会根据 skill 结构识别候选目录。

### 5.3 本地目录、文件和批量目录

- **本地文件夹**：选择一个 skill 目录导入中央库。
- **文件**：导入支持的 `.zip`、`.skill` 等文件。
- **批量导入目录**：扫描父目录，选择其中识别到的多个 skill。

导入是把内容纳入中央库管理。若来源原本位于某个 Agent 目录，导入后仍应检查旧目录是否已成为受管 target，避免把“中央副本”和“原始本地目录”误认为同一对象。

### 5.4 扫描现有 Agent Skills

扫描会列出各 Agent 目录中已存在、但中央库可能尚未管理的 skill。可以单个导入或全部导入。扫描本身只读取；导入会创建中央 skill，并可能建立或关联管理记录。

部分网络安装任务（如 skills.sh 和 Git 安装）会显示进度并允许取消；本地文件、目录扫描和批量目录导入没有统一的取消机制。取消只阻止尚未完成的步骤；已成功写入的条目可能保留，应在任务结束后检查中央库和活动日志。

## 6. 我的 Skills

![我的 Skills](../assets/demo/library.png)

此页面管理中央技能库，不是单个 Agent 的目录视图。

### 6.1 浏览、筛选和预览

- 使用搜索和标签筛选中央 skill。
- 可按来源类型筛选 `local`、`import`、`git` 和 `skills.sh`。
- “未标签”筛选用于找出尚未归类的项。
- 打开详情可预览说明文档和文件信息。
- 标签只影响分类和筛选，不会创建或删除 Agent target。
- 在标签筛选项上打开菜单可全局重命名或删除标签。重命名会替换所有中央 skill 上的旧标签；删除会先确认，再从所有中央 skill 移除该标签。两者都不删除中央文件或任何 target。

### 6.2 更新

有可追踪来源的 skill 可检查或执行更新。更新修改中央 skill 内容：

- `symlink` target 指向中央内容，通常立即反映更新。
- 已登记的全局 `copy` target 会在中央 skill 更新时自动重新同步；项目 copy 不写入全局 target 记录，需要在项目工作区手动更新。
- 无可靠来源或本地修改冲突时，更新可能不可用或失败。

页面支持检查全部来源更新、更新全部可更新项，也支持多选后批量更新。批处理逐项执行，可能部分成功；以结束后的成功、未变化和失败计数为准。

本地或导入来源变为 `source_missing` 时，可重新选择目录以关联新来源；也可“转为独立副本”，即解除外部来源路径，保留当前中央内容，之后不再从原目录重新导入。解除来源不是删除中央 skill 或 Agent target。

更新前建议先完成备份。更新失败时查看活动日志，不要仅根据版本提示判断目录是否已改变。

### 6.3 Preset 成员与 Agent 开关

在 skill 详情中可为 Preset 配置目标 Agent。该开关首先表达“此 skill 是否属于该 Preset 的该 Agent 成员”。

- 编辑 **active Preset** 时，打开 Agent 开关会立即创建对应全局 target，关闭会立即删除。
- 编辑非 active Preset 时，打开或关闭开关都只改 Preset 配置，不改变当前磁盘 target。
- 把整个 skill 加入或移出 Preset 也只改变成员配置，不创建或删除磁盘 target。
- 从 Preset 移除成员不等于删除中央 skill；需要删除 target 时，应使用工作区 PresetBar或明确的单项移除操作。

> **当前实现限制：** 工作区顶部 PresetBar 的状态和点击操作不会读取这些 Agent 开关。它会把 Preset 中的每个 skill 与当前页面的每个 Agent 组合后统一检查、添加或移除。因此，即使某个 skill 的 Agent 开关已关闭，在包含该 Agent 的工作区点击 Preset 仍可能添加它；点击 active Preset 也可能移除该 Agent 中全部属于此 Preset 的 skill。

> **无二次确认：** 在 active Preset中关闭 Agent 开关会立即删除对应全局 target，不会再弹出破坏性确认框。

多选工具栏还可批量把选中 skill 加入/移出当前正在查看的 Preset，以及批量编辑标签。Preset 成员批量操作只修改成员配置，不直接应用工作区 Preset；批量标签只改中央元数据。批处理可能部分成功，完成后刷新并核对计数。

### 6.4 卡片 Agent 图标与项目分配

每张 skill 卡片/列表行显示所有已启用且已安装 Agent 的状态图标。空心图标表示该 Agent 没有 target，点击会立即建立 target；已同步图标再次点击会立即移除对应软链接或 copy 副本。该操作没有二次确认。

这些图标是直接 target 开关，不是详情中的 Preset Agent 开关。若该 skill 属于真正的 active Preset，点击还会同步修改 active Preset 对应 Agent 的 toggle；否则只改 target。移除使用物理路径，多个 Agent 共享 Skills 根时会同时影响其他 Agent并可能留下陈旧记录。

skill 详情的“项目”区块可按已登记项目和 Agent 直接导出中央 skill，也可再次点击已安装状态将项目项删除。两种操作都立即写磁盘；删除没有二次确认，并继承项目真实目录递归删除和共享项目路径风险。它与从项目详情页添加/删除调用相同的项目文件操作。

### 6.5 删除中央 skill

卡片删除、详情删除和批量删除均属于中央库删除。确认后会删除中央 skill，并清理数据库记录的 Agent 全局 target。它与“在某个 Agent 工作区点垃圾桶”不同，影响范围更大。

项目导出项不写入这类全局 target 记录，因此不会随中央 skill 自动清理：项目 copy 会保留为独立副本，项目 symlink 会因中央源被删除而变成断链。删除中央 skill 后，应逐个项目工作区检查并手动处理。

删除中央 skill 前确认：

- 是否仍被多个 Preset 使用；
- 是否在多个 Agent 或项目中有 target；
- 是否有未备份的本地修改；
- target 是软链接还是 copy 副本。

## 7. Preset 状态与点击行为

本节的“点击 Preset”特指全局、Lobster 或项目工作区顶部的 PresetBar。侧边栏 Preset 只切换浏览上下文。

> **无二次确认：** PresetBar按钮点击后立即执行。状态为 `active` 时会直接批量删除当前作用域组合；状态为 `partial` 或 `inactive` 时会直接开始添加。

### 7.1 状态含义

| 状态 | 判断方式 | 界面含义 |
|---|---|---|
| `active` | Preset 中每个 skill 与当前页面每个 Agent 的组合都已存在 | 显示已激活标记；在工作区 PresetBar 点击将移除这些组合 |
| `partial` | 上述组合只存在一部分 | 显示 `installed/total`；在工作区 PresetBar 点击补齐缺失组合 |
| `inactive` | 上述组合一个都不存在 | 在工作区 PresetBar 点击添加全部组合 |
| `empty` | Preset 没有成员，或当前作用域没有目标 Agent | 通常不显示或不可用于同步 |

`installed/total` 当前按“Preset skills × 当前页面 Agents”的笛卡尔积统计，不按“我的 Skills”中为 Preset 配置的 Agent 开关过滤，也不保证等于 Agent 内显示的 skill 数。一个 skill 可对应多个 Agent，目录里也可能存在 Preset 外的 skill。

### 7.2 点击 inactive 或 partial Preset

系统只补齐当前作用域中缺少的成员：

- 保留其他 Preset 已添加的 target；
- 保留独立添加的 target；
- 保留非受管本地 skill；
- 不把当前目录强制裁剪成与 Preset 完全一致。

因此，点击一个包含 10 个成员的 Preset 后，Agent 目录可能仍有 100 个 skill。这是叠加语义，不是计数错误。

### 7.3 点击 active Preset

系统删除该 Preset 在当前 Agent/工作区作用域内的成员 target。它不会删除中央 skill，也不会主动清理 Preset 之外的 target。

若同一个 target 同时由其他业务关系需要，界面最终状态应以刷新后的工作区为准。执行前确认当前页面是“全部 Agents”、单个 Agent 还是项目工作区，因为作用域不同。

### 7.4 正在查看与真正 active

当前存在三个独立状态：

- **正在查看的 Preset**：侧边栏选择，只切换“我的 Skills”的浏览/编辑上下文。
- **active Preset**：后端记录的最近默认全局应用状态，启用 Agent、改路径和部分开关操作会读取它。
- **默认启动 Preset**：设置页保存的选择；修改后不会立即改变 active Preset，下次 GUI 启动才生效。

以下操作尤其依赖真正 active Preset：

- 启用 Agent 后的默认同步；
- 某些 Preset 成员开关的即时 target 清理；
- 路径变更后的重新同步。

设置页只显示默认启动 Preset，不直接显示当前 active Preset。工作区 PresetBar只补齐或移除当前作用域 target，也不会改变后端 active Preset。排查“我正在看 A，为什么同步了 B”时，记录侧边栏选择、默认启动设置和最近一次新建、删除或 GUI 启动作；无法据此推断 active 状态时，不执行批量删除。

### 7.5 新建 Preset

> **高风险且无额外确认：** 当前实现中，新建 Preset 不只是创建一份空配置。创建对话框提交后会立即取消同步原 active Preset，再将新建的空 Preset 设为 active，不会针对批量删除再弹一次确认。因此，仅点击“新建 Preset”就可能批量删除原 active Preset 中各 skill 的全部受管全局 target。target 记录不区分由哪个 Preset或独立操作创建，同一 skill 即使也属于其他 Preset，也可能被删除。

新建前记录正在查看的 Preset、默认启动 Preset、最近一次 Preset动作和各 Agent工作区现状；无法推断 active 状态时不要新建。新建完成后，侧边栏重新选择原 Preset只能恢复浏览上下文，不能恢复 active 状态或 target。需要立即恢复文件时，在对应工作区用 PresetBar补齐；需要恢复后端 active 状态时，把原 Preset设为默认启动 Preset并重启 GUI。

### 7.6 重命名与排序

侧边栏可重命名 Preset、修改图标并拖拽排序；“我的 Skills”中也可调整当前 Preset 的成员显示顺序。这些操作只改 Preset 元数据或成员顺序，不直接新增、删除 target。

但侧边栏 Preset 排序会决定删除 active Preset 后的自动切换对象：系统按持久化顺序选择剩余首项，设为新的 active Preset并立即同步。执行 active Preset 删除前，先确认侧边栏首个剩余 Preset及其成员范围。

### 7.7 删除 Preset

- 删除 active Preset：会删除其中各 skill 的全部受管全局 target，再删除 Preset 配置；若还有其他 Preset，系统会把剩余列表第一项设为 active，并立即同步该 Preset，因此可能同时批量新增另一组 target。
- 删除 inactive Preset：删除 Preset 配置，但不会把当前磁盘上同名成员当作残留自动清理。
- 两种情况都不等于删除中央 skill。

## 8. 全局工作区

![全局工作区](../assets/demo/global-workspace.png)

![单个 Agent 工作区](../assets/demo/agent-workspace.png)

全局工作区直接反映 Agent 用户级 Skills 目录。可进入“全部 Agents”总览，或选择单个 Agent。

### 8.1 添加 Skills

点击“添加 Skills”，从中央库选择 skill，并选择目标 Agent。提交后建立受管 target：

- 在 `symlink` 模式下创建软链接；
- 在 `copy` 模式下复制目录或文件；
- 不会自动把 skill 加入 Preset，除非界面明确执行了 Preset 编辑。

“全部 Agents”中的一次选择可能产生多个 target。单个 Agent 页面只请求修改该 Agent 的管理记录；若多个 Agent 共享物理 Skills 根，新增内容也可能被其他 Agent 看见。

### 8.2 移除单个受管 target

工作区卡片上的垃圾桶只移除当前 Agent 目录中的 target 和对应管理记录：

- 不删除中央 skill；
- 不主动删除项目/其他工作区的管理记录；物理路径重合时，共享内容仍会消失；
- `symlink` 模式删除软链接；
- `copy` 模式删除目标副本。

受管 skill 的垃圾桶没有二次确认，点击后立即删除当前 Agent 记录对应的物理 target。若其他 Agent 共用同一路径，该文件会同时消失，但其他 Agent 的 target 记录可能仍保留；刷新时可能出现计数或磁盘状态不一致。同一风险也适用于全局工作区 PresetBar 按 Agent 移除和“我的 Skills”中 active Preset 的 Agent 开关。

### 8.3 删除非受管本地 skill

工作区也会显示不是由 Skills Manager 创建的本地内容。对这类项执行删除时，系统面对的是 Agent 目录里的真实文件系统对象：

- 对象可能是软链接；
- 也可能是普通文件或真实目录；
- 删除后中央库通常没有副本可恢复。

非受管本地 skill 的删除会显示名称和 Agent 确认，但不会展示完整路径。不确定时，先导入中央库或在文件管理器中检查路径和链接类型，再删除。

### 8.4 本地项与中央库双向操作

单个 Agent 工作区对扫描到的本地项还提供以下操作：

- **导入/上传到中央库**：方向为 Agent 本地目录 → 中央库。`project_only` 本地项会直接创建中央 skill；已有中央对应项但内容不一致时会先确认，随后更新中央内容。
- **从中央拉取**：方向为中央库 → 当前 Agent 本地目录，会写入或覆盖本地内容。界面提供危险确认；本地版本更新时后端会阻止不安全覆盖。
- **移除受管 target**：删除当前 Agent 的管理记录及其对应物理项，无二次确认；共享路径风险仍适用。
- **删除非受管本地项**：删除当前 Agent目录中的软链接、文件或真实目录，有名称确认，但中央库通常无法恢复。

每次操作后根据 `project_only`、`project_newer`、`center_newer`、`diverged` 或 `in_sync` 状态判断方向。`diverged` 表示两边都有变化，先预览和备份，不能把“上传”与“拉取”混用。

## 9. Lobster Agent 工作区

Lobster Agent 工作区使用与全局工作区相同的核心模型：每个 Agent 有独立的配置和工作区视图，可添加、移除、扫描和通过 Preset 批量操作；不同 Agent 的物理目录仍可能共享。它被单独分组是为区分 Agent 类型，不代表 target 会写入编码 Agent 的目录。

操作前确认页面标题和目标 Agent。Lobster 与编码 Agent 即使显示同名 skill，也可能有两套独立管理记录；删除一方不会主动清理另一方的记录，但物理路径重合时共享内容仍会消失。

## 10. 项目工作区

![项目工作区](../assets/demo/project-workspace.png)

项目工作区管理 `<project>` 下各 Agent 的项目相对目录。它与全局工作区互不替代：

- 全局 target 对该用户运行的所有项目可见；
- 项目 target 只对目标项目可见；
- 同一 skill 可同时存在全局和项目 target；
- 项目删除不会主动清理全局 target 记录；但关联工作区可指向任意外部目录。若项目/关联工作区路径与全局或其他工作区物理重合，删除项目项仍会让共享目录中的内容消失。

### 10.1 登记项目

“添加项目”提供三种方式：

- **手动选择**：选择一个项目目录并登记。
- **扫描目录**：扫描一个根目录，勾选一个或多个识别到的项目后批量登记；重复项会被跳过。
- **关联工作区**：把现有外部 Skills 根目录作为独立工作区登记。它只作为该工作区管理，不参与普通全局 target 或全局 Preset 同步；不要把它当作某个项目根目录。

关联工作区会尝试在 Skills 根目录旁创建同名加 `-disabled` 后缀的目录。启用/禁用操作在启用根与该 disabled 根之间移动内容；若 disabled 目录无法创建，该工作区可能不支持启停。两个根必须彼此独立，不能相同或互相嵌套。

添加后，应用保存路径和可用 Agent 信息。登记本身不要求立刻写入 skill。移除项目登记只移除应用中的项目记录，不等同于删除整个项目目录。

### 10.2 添加和分配

“添加 Skills”弹层可选择中央 skill，并为项目选择一个或多个目标 Agent。每个目标 Agent 使用设置中的项目相对路径；自定义 Agent 未配置项目路径时，可能无法作为项目目标。

### 10.3 启用、双向更新和标签

项目卡片和批量工具栏支持：

- **启用/禁用**：在项目的启用目录与 disabled 目录之间移动 skill，不删除中央 skill。该操作是否显示取决于项目 Agent 是否支持目录切换。
- **项目 → 中央**：`project_only` 会导入中央库；`project_newer` 或 `diverged` 会用项目内容更新中央 skill。
- **中央 → 项目**：`center_newer` 或允许处理的 `diverged` 可用中央内容更新一个或多个项目 Agent变体。`project_newer` 会被后端保护性拒绝，需先执行项目 → 中央或手工处理差异。
- **批量更新**：逐项执行同一方向，可能部分成功；`project_newer` 出现在中央 → 项目批次时会计为失败。结束后查看成功/失败计数并刷新。
- **标签**：项目中已关联中央 skill 的项可批量修改中央标签；`project_only` 项需先导入中央库。

这些启用、更新和 Agent开关按钮点击后直接执行，没有统一的二次确认。项目 → 中央可能覆盖中央内容，中央 → 项目可能覆盖项目内容；操作前根据同步状态确认方向并备份。

### 10.4 删除和批量删除

在项目详情中移除 Agent、删除 skill 或批量删除，会直接删除该项目 Agent Skills 根目录内选中的项。它不会删除中央 skill，也不会主动删除其他项目或全局 target 的管理记录；物理路径重合时仍会影响共享目录。

> **高风险：** 项目列表同时包含受管项和 `project_only` 等非受管本地项。项目删除命令不要求中央库匹配；选中项若是真实目录，会被递归删除。先检查对象类型并备份，不能把项目删除理解成“只删软链接”。

显式“删除 skill”和“批量删除”会显示名称或数量确认；详情中的 Agent开关关闭、项目工作区 PresetBar点击 active Preset则没有二次确认，会立即调用项目删除。

若同一项目的多个 Agent 解析到相同 Skills 路径，关闭一个 Agent 开关或删除其 variant 会删除共享物理项，其他 Agent 也会失去该 skill并可能留下陈旧记录。批量操作对同一路径重复删除时还可能部分失败；先比较项目中各 Agent 的解析路径并在完成后刷新核对。

删除整个项目登记前，如需保留磁盘内容，应先阅读确认框；项目登记与项目磁盘数据是两个概念。

## 11. 添加 Skills 弹层

添加弹层的两个选择维度：

1. **选哪些中央 skill**；
2. **写到哪些目标 Agent**。

作用范围由打开弹层的页面决定：全局工作区写用户级目录，项目工作区写项目相对目录。批量选择会生成多个 target；部分失败时可能出现一部分成功、一部分失败。关闭提示前记录失败项，刷新页面核对实际结果。

## 12. 备份与多设备同步

![备份与多设备同步](../assets/demo/backup.png)

备份页把中央技能库连接到 Git 远端，用于版本历史、恢复和多设备同步。Agent target 通常可由中央库和配置重建；备份重点是中央内容、元数据和同步状态。

### 12.1 连接与同步

- 可通过 GitHub 登录连接备份仓库，或在设置中配置 Git 远端。缺少同名仓库时会创建私有仓库；同名既有仓库会直接复用，可能是公开仓库。
- 多设备连接同一仓库后，变更按 skill 合并。
- 可编辑设备名称；它用于备份提交作者和多设备历史识别，不会更改 Agent 名称。
- 自动备份开关控制后台自动备份；关闭后仍可手工立即备份。自动备份失败会在页面保留错误状态，直到后续备份成功。
- 同一 skill 的真实冲突不会静默覆盖；需在界面选择“保留本地”“使用远端”或“保留两者”。每种选择前都会创建安全快照；“保留两者”会保留两个可区分副本，需后续整理名称和 Preset 成员关系。
- 网络、认证、代理或 Git 状态异常时，同步可能部分失败；先保留本地内容并查看错误。

连接公开仓库时界面会显示限时警告。公开仓库会暴露 skill 文件、标签、Preset及其成员关系；看到警告后应立即到 GitHub 核对仓库可见性，非有意公开时改为私有或断开连接。

本机尚未初始化备份仓库时，首次设置有两条路径：

- **使用已有远程备份**：适合第二台及后续设备。系统临时移开本机现有中央 skill，克隆远端，再把不冲突的本机内容合并回来；不能把它理解为直接丢弃本机内容。
- **把本机作为新的备份源**：只在远端为空或应以本机内容为准时使用。系统初始化本地 Git 并设置远端；完成后再执行一次“立即备份”，才会提交、合并并推送当前内容。

出现 unrelated histories、detached HEAD 或普通 Git 冲突等恢复状态时，界面可能提供“使用远程备份恢复本机（重新克隆）”。该操作会暂存原 `.git` 和本机 skill，使用远端历史重新克隆，再合并不冲突的本机内容；成功后原 `.git` 临时备份会被删除。失败时系统会尝试恢复原文件和原 `.git`。若错误提示无法恢复并给出 `skills-git-recovery-*` 或 `skills-backup-before-clone` 路径，不要删除这些同级恢复目录；先停止再次同步、导出日志并检查当前中央库和 Git 状态。界面中的“保留本机内容，建立跟踪”当前只给出终端操作提示，不是另一个自动 GUI 恢复流程。

### 12.2 快照恢复

恢复快照会改变中央库到所选历史状态。系统会先把未提交改动提交为“恢复前备份”，再为恢复前状态创建可见的 safety-point 快照，然后以新提交应用历史内容；因此可从历史再次恢复到操作前状态，无需用户额外手工创建恢复前备份。`symlink` target 可能随中央内容立即变化；`copy` target 仍需重新同步。恢复后核对 Preset、标签和各工作区 target。

### 12.3 三种断开/删除操作

| 操作 | 本机连接 | GitHub 授权 | 远端仓库数据 |
|---|---|---|---|
| 断开本机 | 删除/停用本机连接 | 保留 | 保留，其他设备不受影响 |
| 打开 GitHub 授权设置并断开本机 | 立即断开本机备份连接 | GUI 不直接撤销；必须在 GitHub 页面手工完成 | 保留 |
| 打开远端备份删除页 | 不立即改变 | 不立即改变 | GUI 只打开 GitHub 仓库 Danger Zone；必须在 GitHub 完成输入仓库名等确认后，远端仓库才会被永久删除 |

“撤销 GitHub 授权”入口只打开 GitHub OAuth/PAT 设置页，并立即断开本机连接；未在 GitHub 页面手工撤销时，授权本身仍可能有效。断开本机也不等于删除远端备份。GUI 的删除入口本身不调用删除 API；未在 GitHub Danger Zone 完成确认，就没有删除远端仓库。最终删除是不可逆的外部操作，应先确认其他设备和必要快照已迁移。

## 13. 设置

![设置](../assets/demo/settings.png)

### 13.1 中央库路径

可打开中央库目录、覆盖默认路径或恢复默认路径。改变中央库位置会改变安装、更新、备份和 target 来源。界面提示需要重启时，应重启后再操作，避免旧进程继续引用旧路径。

### 13.2 Agent 路径与自定义 Agent

- 可覆盖内置 Agent 的全局 Skills 路径。
- 可设置自定义 Agent 的名称、全局路径和项目相对路径。
- 可恢复默认路径。
- 删除自定义 Agent 会删除其受管 target 和平台配置，不删除中央 skill。

改变或重置 Agent 全局路径时，系统会清理旧路径中的受管 target，再按 active Preset 向新路径同步。保存路径、恢复默认和删除自定义 Agent 都没有共享路径保护；旧路径被其他 Agent 使用时，其物理 target 也可能被删除。上述操作没有二次确认，点击后立即执行；操作前先记录并比较所有 Agent 的旧路径、新路径和工作区数量并完成备份。旧路径中的非受管内容不应被当作受管 target 清理。

项目相对 Skills 路径的保存/重置语义不同：它只更新 Agent 配置，不迁移、删除或重建任何已登记项目中的现有内容，也没有二次确认。保存后，项目工作区立即改为扫描和写入新相对路径；旧路径内容仍留在磁盘，但可能从 GUI 当前视图消失。改动前记录旧路径和涉及项目，之后需手工迁移或分别清理，不能套用上一段全局路径的自动 reconcile 语义。

### 13.3 启用或禁用 Agent

- 禁用单个平台：删除该平台全部受管 target，并将平台标记为禁用。
- 重新启用：按 active Preset 重新同步受管 target。
- 禁用全部平台：删除所有平台的受管 target。
- 禁用不删除中央 skill，也不等于删除 Preset 配置。

这也是 GUI 中“一次删除指定平台全部受管软链接”的主要操作。由于底层按 target 删除，若同步模式为 `copy`，实际删除的是副本目录/文件，不只是软链接。

> **高风险：** 当前单个平台和全部平台的启用/禁用开关没有二次确认。点击禁用会立即开始批量删除受管 target；单个平台与其他平台共享路径时，还可能删除其他平台正在使用的物理项并留下陈旧记录。操作前先比较各平台路径并记录工作区数量。

### 13.4 默认启动 Preset

设置页修改默认启动 Preset时，只保存 `default_scenario`，不会立即改变 active Preset或磁盘 target。下次 GUI 启动时，若它与当前 active Preset不同，系统会先取消同步原 active Preset的全部受管全局 target，再把默认启动 Preset设为 active并同步其配置。

这是条件性的批量删除和新增操作。修改后不要只看侧边栏高亮判断结果；重启前记录正在查看的 Preset、旧默认启动 Preset、最近一次 Preset动作和各 Agent工作区现状。无法推断 active 状态时不要重启生效，重启后再次检查各 Agent工作区。

### 13.5 同步模式

- `symlink`：节省空间，中央更新即时可见；依赖系统支持软链接和路径稳定。
- `copy`：目标独立、兼容性较好；占用更多空间，更新需重新同步。

切换模式影响后续同步。不要假设现有 target 会自动全部转换；切换后逐个工作区核对类型和内容。

### 13.6 外观和桌面行为

可设置主题、字号、语言、托盘图标和关闭窗口后的行为。关闭行为设为隐藏时必须保留托盘入口；禁用托盘图标会阻止不可恢复的“隐藏但无入口”组合。

### 13.7 网络、Git 和自动更新

- 代理地址影响需要联网的市场、Git 和更新请求。
- Git 远端用于备份与同步。
- 可选择 Git/合并相关实现。
- 自动更新包含检查频率和自动应用策略；应用更新与 skill 内容更新是不同流程。

### 13.8 诊断

- **刷新 Agent**：重新检测工具和路径状态。
- **导出日志**：把最近日志和活动记录打包为压缩文件。
- **报告 Issue**：收集诊断信息并打开反馈入口；提交前检查内容是否含不希望公开的本机路径。
- **打开中央库目录**：直接检查中央 skill 文件。

## 14. 删除与停用影响矩阵

| GUI 操作 | 删除/改变什么 | 不影响什么 | 风险 |
|---|---|---|---|
| Agent 工作区删除受管 skill | 当前 Agent 的单个 target 和记录 | 中央 skill、其他工作区的管理记录 | **无二次确认；路径重合时会影响项目/其他 Agent并可能留下陈旧记录；`copy` target 会删除副本** |
| Agent 工作区删除非受管本地 skill | 当前路径中的软链接、文件或真实目录 | 通常无中央备份 | **高：可能永久删除真实目录** |
| 工作区 PresetBar 点击 active Preset | 当前作用域内该 Preset 的成员 target | 中央 skill、Preset 外 target | **无二次确认；作用域可能是单 Agent、全部 Agent或项目**；侧边栏点击不执行此操作 |
| 工作区 PresetBar 点击 partial/inactive Preset | 新增缺失 target | 已有其他 target | 不执行“精确匹配”，总数可大于 Preset；侧边栏点击不执行此操作 |
| My Skills 关闭 active Preset 的 Agent 开关 | 对应成员配置和即时 target | 中央 skill | **无二次确认；只在 active Preset下即时清理；共享路径时会影响其他 Agent** |
| My Skills 关闭非 active Preset 的 Agent 开关 | Preset 配置 | 当前磁盘 target | 容易误以为 target 已删除 |
| My Skills 卡片/列表 Agent 图标 | 当前 skill 的单 Agent target；属于 active Preset时也改对应 toggle | 中央 skill | **无二次确认；移除时共享路径会影响其他 Agent** |
| My Skills 详情的项目 Agent 状态 | 向所选项目 Agent 导出或删除该项目项 | 中央 skill、其他工作区的管理记录 | **无二次确认；路径重合时可影响全局/其他工作区物理内容；真实目录删除风险同项目详情页** |
| 删除标签 | 从所有中央 skill 移除该标签 | 中央文件、target、Preset | 有确认；这是全局元数据操作，不是删除 skill |
| My Skills 删除/批量删除 | 中央 skill及其数据库记录的 Agent 全局 target | 项目目录项不会被自动删除 | **高：项目 copy 会残留，项目 symlink 可能断链** |
| 新建 Preset | 创建空 Preset、删除原 active Preset中各 skill 的全部受管全局 target、把新 Preset设为 active | 中央 skill、项目目录 | **无额外破坏性确认；同一 skill由其他 Preset或独立操作创建的全局 target也可能被删** |
| 删除 active Preset | Preset 配置及其中各 skill 的全部受管全局 target；按侧边栏持久化排序选择剩余首个 Preset，设为 active并立即同步 | 中央 skill、项目目录 | **高：拖拽顺序决定自动同步对象；可能同时批量删除旧 target、新增另一组 target** |
| 删除 inactive Preset | Preset 配置 | 现有磁盘 target、中央 skill | 可能留下仍在使用的 target |
| 设置中禁用单个平台 | 该平台全部受管 target；平台状态变为禁用 | 中央 skill、Preset 配置 | **共享路径时会影响其他平台并可能留下陈旧记录；`copy` 模式删除全部副本** |
| 设置中禁用全部平台 | 所有平台受管 target | 中央 skill、Preset 配置 | **高：全局批量清理** |
| 删除自定义平台 | 该平台受管 target和平台配置 | 中央 skill | **共享路径时会影响其他平台；删除前比较所有平台路径** |
| 改变/重置 Agent 路径 | 清理旧路径受管 target；新路径同步 active Preset | 中央 skill | **无二次确认；共享旧路径时会影响其他平台；路径填错会立即建立 target** |
| 改变/重置项目相对路径 | 只更新项目扫描/写入配置 | 旧项目路径中的现有内容 | **无二次确认；不自动迁移，旧内容可能从 GUI 消失但仍留盘** |
| 修改默认启动 Preset并重启 GUI | 若新默认与当前 active 不同，删除旧 active Preset中各 skill 的受管全局 target；同步新默认 Preset并设为 active | 中央 skill、项目目录 | **高：条件满足时，重启触发跨 Agent批量删除和新增** |
| 项目显式删除 skill/批量删除 | 直接删除当前项目 Agent Skills 根目录内选中项 | 中央 skill、其他工作区的管理记录 | **路径重合时可影响全局/其他工作区；重复删除可能部分失败；真实目录可能被递归删除** |
| 项目关闭 Agent开关或 PresetBar移除 | 直接删除当前项目对应项 | 中央 skill、其他工作区的管理记录 | **无二次确认；路径重合时可影响全局/其他工作区；真实目录可能被递归删除** |
| 移除项目登记 | 应用中的项目记录 | 整个项目目录本身 | 不等于磁盘清理 |
| 断开备份 | 本机备份连接 | 远端仓库、其他设备 | 不会释放远端数据 |
| 打开 GitHub 授权设置并断开本机 | 本机连接 | 未在 GitHub 手工完成前，授权和远端仓库不变 | GUI 不直接撤销 OAuth/PAT授权 |
| GUI 打开远端备份删除页，并在 GitHub 完成确认 | GitHub 远端仓库 | 本机当前中央库 | GUI 只负责打开 Danger Zone；GitHub 最终确认后才删除，且不可逆 |

## 15. Preset 数量与 Codex 显示不一致

出现“Preset 有 N 个成员，Codex 显示的启用 skills 不是 N 个”时，按以下顺序检查：

1. **确认 Preset 作用域。** 工作区 PresetBar 会把 Preset 中的 skill 与当前页面 Agents 组合；单个 Codex 页面只看 Codex，“全部 Agents”页面会产生跨 Agent 组合。
2. **确认点击前状态。** `partial` 点击是补齐，`active` 点击是移除，含义相反。
3. **理解叠加语义。** 应用 Preset 不删除其他 Preset 或独立添加的 target。
4. **检查 Codex 全局目录。** 目录可能含非受管本地 skill、旧 target 或其他根目录中的 skill。
5. **分清三个 Preset 状态。** 侧边栏显示正在查看的 Preset，设置页显示默认启动 Preset；两者都不能直接代表后端当前 active Preset。
6. **检查平台路径覆盖。** GUI 和 Codex 可能扫描不同目录。
7. **检查平台启用状态。** 禁用后 target 会被删除；重新启用按 active Preset 重建。
8. **确认同步模式。** 已登记的全局 copy 会在中央更新时重同步，但项目 copy 和单纯的 Preset 成员编辑不会随之改变磁盘内容。
9. **刷新和重启。** 刷新 GUI 工作区，并重启 Codex 会话，排除运行时缓存。
10. **查看活动日志。** 查找部分失败、权限问题、同名目标或无效软链接。

关键结论：当前 Preset 点击逻辑是“补缺/移除本 Preset”，不是“把 Codex 精确变成该 Preset”。若需要精确匹配，必须先明确哪些额外 target 可以删除；GUI 当前不会在点击另一个 Preset 时自动删除所有多余链接。

## 16. 软链接被删后又出现

常见原因：

- 重新启用了 Agent，系统按 active Preset 自动同步；
- 改变或重置了 Agent 路径，系统向新路径同步 active Preset；
- 再次点击了 inactive/partial Preset；
- 从“添加 Skills”弹层重新添加；
- 多设备同步恢复了中央配置，随后触发 target 重建；
- 删除的是非 active Preset 成员配置，磁盘 target 当时并未删除。

GUI 当前不直接显示后端 active Preset。先记录侧边栏正在查看的 Preset、设置中的默认启动 Preset，以及最近一次新建、删除或启动动作；无法判断 active 状态时，不要执行批量删除，先在单个 Agent 工作区移除一个 target并观察。只有明确要清空该 Agent全部受管 target时才禁用 Agent。只删除文件管理器中的软链接可能留下管理记录，后续同步会重建。

## 17. 故障排查

### 17.1 工作区显示 partial

刷新后仍为 partial，通常表示部分组合创建失败、目标 Agent 被禁用、路径不可写或目标同名对象冲突。PresetBar 不按详情中的 Agent 开关过滤；先确认当前页面包含哪些 Agents，再查看 `installed/total` 和活动日志。

### 17.2 添加或删除部分成功

批量操作不是“全有或全无”的保证。网络、权限或单个路径错误可能只影响部分项。不要直接重复整批删除；先刷新，确认已成功项，再重试失败项。

### 17.3 无效软链接

中央库被手工移动、删除或改名后，旧软链接可能失效。优先通过设置修改中央库路径，或在 GUI 中移除旧 target 后重新添加。不要同时手工移动中央目录和批量点击 Preset。

### 17.4 同名目标冲突

目标目录已有同名真实目录时，系统不应无提示覆盖。先判断该目录是否为非受管本地 skill；需要保留时先导入或备份，再处理冲突。

### 17.5 Agent 未显示或不可用

在设置中检查：

- Agent 是否已安装并被检测到；
- 是否被禁用；
- 全局路径是否正确；
- 自定义 Agent 是否配置了名称和全局路径；
- 用于项目时是否配置项目相对路径。

点击刷新 Agent 后再检查。仍异常时导出日志。

### 17.6 备份或安装联网失败

检查代理地址、Git 远端、GitHub 授权和网络连通性。失败后保留本地中央库；不要为解决同步问题手工删除中央目录。

## 18. 活动日志、导出日志与反馈

应用记录安装、移除、更新和同步等活动。排查时建议保留以下信息：

- 操作时间和页面；
- 目标 Agent 或项目；
- Preset 名称及点击前状态；
- `installed/total`；
- 同步模式；
- 实际全局/项目路径；
- 错误提示原文；
- 刷新后仍存在的 target 列表。

在 **设置 → 导出日志** 生成压缩包，再通过 **报告 Issue** 反馈。日志可能包含本机路径、仓库 URL 和诊断环境信息；公开上传前应检查并脱敏。

## 19. 删除前快速检查

执行任何批量删除、平台禁用、路径修改或中央 skill 删除前，确认：

- 当前页面是全局、单 Agent、Lobster 还是项目工作区；
- 对象是中央 skill、受管 target 还是非受管本地内容；
- target 是 `symlink` 还是 `copy`；
- 已分别记录正在查看的 Preset和默认启动 Preset；GUI 不直接显示 active Preset，无法从前两者确定时不执行批量删除；
- 是否会跨 Agent、跨项目或跨设备；
- 中央库和远端备份是否可恢复；
- 已从设置页记录目标 Agent路径，并在工作区核对对象和数量；若确认框提供名称或数量，再与记录交叉检查。

无法确认时，先导出日志并备份中央库，再从单个 Agent 的单个 target 开始验证影响范围。
